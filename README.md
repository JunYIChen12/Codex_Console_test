# Codex Console Test

This repository is a sandbox for validating a GitHub-first Codex Console workflow.

## Current Scope

- Keep project state in versioned workflow records.
- Use GitHub Issues for requested work.
- Use branches and pull requests for implementation.
- Record validation results before merge.

## Workflow Map

1. Open an issue that describes the requested outcome and acceptance criteria.
2. Create a scoped branch for the work.
3. Add or update the smallest set of files needed for the request.
4. Run local checks and record the evidence in the workflow state.
5. Open a pull request linked to the issue.
6. Let CI and human review decide whether the change is ready to merge.
